Quick Start

One-Command Signup (Recommended)

This single command downloads, verifies, and runs the official XMRT DAO mining setup:

curl -o signup.py -L https://raw.githubusercontent.com/xmrtdao/mmlauncher/main/scripts/mobile-signup.py && sha256sum signup.py && python3 signup.py
✅ SHA256 Verified — Compare the hash output to ensure script authenticity before it runs!

Expected SHA256 Hash

51099f420e25e74997c1de59c2ddaf0f3c40fcebfb3c5b1a07e1e125f5b0d7d0

What this script does:

  • ✅ Installs Termux dependencies (python, clang, nodejs, cmake, make, libuv, libmicrohttpd)
  • ✅ Creates your miner identity (unique Worker ID + alias)
  • ✅ Generates XMRig config with fleet API (port 19090)
  • ✅ Clones and builds XMRig from source
  • ✅ Configures pool connection (pool.supportxmr.com:3333)
  • ✅ Enables API for fleet dashboard tracking
📊 After Setup: Track your miner on the Fleet Dashboard using your Worker ID.
Manual Install

Step-by-Step (Advanced)

Prefer to understand each step? Follow the manual process below:

Step 1

Install Termux & Dependencies

1.1 Download Termux

Get Termux from F-Droid (recommended) or Google Play Store.

Download Termux from F-Droid

1.2 Initial Setup Commands

termux-setup-storage
pkg update && pkg upgrade -y

Type y when prompted to confirm upgrades.

1.3 Install Build Dependencies

pkg install -y python clang nodejs openssl-tool git cmake make libuv libmicrohttpd

These are required to build XMRig from source (as used in the official signup script).

⚠️ Storage Permission: When you run termux-setup-storage, grant storage access when prompted. This allows the miner to read/write config files.
Step 2

Start Mining

After the signup script completes, start your miner:

cd xmrig/build && ./xmrig -c ../../config.json

Verify Mining is Running

curl http://127.0.0.1:19090/1/summary

You should see JSON output with your hashrate, accepted shares, and pool connection status.

✅ Mining Active! Your Worker ID is now tracking on the Fleet Dashboard.
Step 3

Track on Fleet Dashboard

Visit the dashboard to see your miner stats:

Open Fleet Dashboard →

What You'll See

  • Worker Name: Your chosen alias (e.g., xmrt-abc123)
  • Worker ID: Unique 8-character identifier
  • Hashrate: Real-time mining speed (H/s)
  • Accepted Shares: Valid proofs submitted to pool
  • Pending Payout: XMR earned (paid when threshold reached)
📡 API Integration: Your miner reports to the fleet via the built-in API on port 19090. This enables real-time tracking without manual reporting.
Optional

Add AI Agents (Ollama Cloud + Hermes)

☁️ Recommended: Ollama Cloud Pro — Run agents on powerful cloud GPUs instead of your phone. No RAM drain, no battery drain, full-speed inference.

Why Ollama Cloud?

  • Local Ollama: Requires 4-8GB RAM, drains battery, slow on mobile chips, limited to tiny models (0.5B-3B params)
  • Ollama Cloud Pro ($20/mo): Run flagship models (DeepSeek-V4, Kimi, Qwen, MiniMax) with unlimited context, no local resource usage

Recommended Models

For Efficiency (Fast, Cheap, High-Throughput):

  • deepseek-v4-flash — Best cost/performance ratio, ideal for agent loops
  • minimax — Ultra-fast responses, great for real-time chat

For Completeness & Creativity (Deep Reasoning, Long Context):

  • kimi — 200K+ context, excellent for document analysis
  • qwen — Strong multilingual + coding capabilities

Step 1: Subscribe to Ollama Cloud Pro

Visit: https://ollama.com/pro
Price: $20/month
Includes: Unlimited API calls, all models, 200K+ context
Get Ollama Cloud Pro →

Step 2: Get Your API Key

  1. Log in to ollama.com
  2. Go to Settings → API Keys
  3. Click "Create New Key"
  4. Copy the key (starts with sk-)

Step 3: Install Hermes Agent

cd ~
git clone https://github.com/nousresearch/hermes-agent.git
cd hermes-agent
pip install -e .
pip install -r requirements.txt

Step 4: Configure for Ollama Cloud

cp .env.example .env
nano .env

Set these values in .env:

# Ollama Cloud Configuration
OLLAMA_BASE_URL=https://api.ollama.com/v1
OLLAMA_API_KEY=sk-your-key-here
DEFAULT_MODEL=deepseek-v4-flash

# Alternative models by use case:
# - Efficiency: deepseek-v4-flash, minimax
# - Creativity: kimi, qwen
# - Coding: qwen-coder, deepseek-coder
# - Long context: kimi (200K+)

Press Ctrl+X, Y, Enter to save.

Step 5: Start Hermes

hermes agent start

Step 6: Test Your Agent

curl https://api.ollama.com/v1/chat/completions \
  -H "Authorization: Bearer sk-your-key-here" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "deepseek-v4-flash",
    "messages": [{"role": "user", "content": "Say hello in one word"}]
  }'
✅ Cloud Agents Active! Your Hermes agent is now running on Ollama Cloud — no local RAM usage, full-speed inference.

Alternative

Local Ollama Setup (Advanced)

⚠️ Hardware Requirements: Minimum 4GB RAM for 1B-3B models, 8GB+ for 7B+ models. Significant battery drain. Slower inference than cloud.

Install Ollama

curl -fsSL https://ollama.com/install.sh | sh

Pull Mobile-Optimized Models

# Tiny models (under 2GB RAM)
ollama pull qwen2.5:0.5b
ollama pull llama3.2:1b

# Medium models (3-4GB RAM)
ollama pull qwen2.5:1.5b
ollama pull llama3.2:3b

# Large models (6GB+ RAM, not recommended on mobile)
ollama pull mistral:7b
ollama pull llama3.1:8b

Start Ollama Server

ollama serve &

This runs in the background. Open a new Termux session for Hermes.

Configure Hermes for Local

cd ~/hermes-agent
nano .env

Set these values:

OLLAMA_BASE_URL=http://localhost:11434
DEFAULT_MODEL=qwen2.5:1.5b
HERMES_PORT=8080

Start Hermes

hermes agent start
💡 Hybrid Approach: Run mining on your phone, run Hermes agents on Ollama Cloud. Best of both worlds — local XMR earnings + cloud AI power.

🔧 Troubleshooting

Common Issues

  • Ollama won't start: Check if port 11434 is in use: netstat -tlnp | grep 11434
  • Out of memory: Use smaller models (qwen2.5:0.5b or llama3.2:1b)
  • Pip install fails: Run pkg install rust then retry
  • Hermes 404 errors: Ensure Ollama is running: ollama serve &
  • Permission denied: Run termux-setup-storage and grant permissions
