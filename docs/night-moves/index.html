Deploy Ollama + Hermes AI Agents on Android/Termux — Complete Step-by-Step Guide

⚡ Complete Guide — From fresh Termux install to running AI agents. No prior experience required.
Phase 1

Install & Configure Termux

1.1 Download Termux

Get Termux from F-Droid (recommended) or Google Play Store.

Download Termux from F-Droid

1.2 Initial Setup

Open Termux and run these commands one at a time:

termux-setup-storage
pkg update
pkg upgrade

When prompted, type y and press Enter to confirm.

1.3 Install Required Packages

pkg install python git curl wget nano termux-api termux-services
pkg install proot proot-distro
pkg install build-essential clang cmake libjpeg-turbo libwebp libtiff libffi zlib

Grant permissions when prompted:

termux-setup-storage
Phase 2

Set Up Python Environment

2.1 Install Python and Pip

pkg install python
pip install --upgrade pip setuptools wheel

2.2 Create Virtual Environment

mkdir -p ~/hermes-env
cd ~/hermes-env
python -m venv venv
source venv/bin/activate

You should see (venv) in your prompt, indicating the virtual environment is active.

2.3 Install Python Dependencies

pip install requests colorama aiohttp fastapi uvicorn[standard]
pip install python-dotenv pyyaml rich click
pip install supabase pydantic python-jose[cryptography]
pip install passlib[bcrypt] python-multipart
⚠️ Note: Installation may take 5-10 minutes on mobile data. Be patient!
Phase 3

Install Ollama for Local AI

3.1 Download Ollama Binary

curl -fsSL https://ollama.com/install.sh | sh

If the script fails, manually install:

curl -LO https://github.com/ollama/ollama/releases/download/v0.1.38/ollama-linux-arm64.tgz
tar -xzf ollama-linux-arm64.tgz
sudo mv ollama /usr/local/bin/
chmod +x /usr/local/bin/ollama

3.2 Start Ollama Server

ollama serve &

Keep this running in the background. Open a new Termux session for the next steps.

3.3 Pull AI Models

For Hermes agents (recommended for mobile):

ollama pull qwen2.5:0.5b
ollama pull qwen2.5:1.5b
ollama pull llama3.2:1b

For more capable agents (requires 4GB+ RAM):

ollama pull qwen2.5:3b
ollama pull llama3.2:3b
ollama pull mistral:7b

3.4 Verify Installation

ollama list
ollama run qwen2.5:0.5b "Hello, are you working?"
Phase 4

Install Hermes Agent

4.1 Clone Hermes Repository

cd ~
git clone https://github.com/nousresearch/hermes-agent.git
cd hermes-agent

4.2 Install Hermes Dependencies

pip install -e .
pip install -r requirements.txt

4.3 Configure Environment

cp .env.example .env
nano .env

Edit the following values in .env:

# Required Configuration
OLLAMA_BASE_URL=http://localhost:11434
DEFAULT_MODEL=qwen2.5:1.5b

# Optional: Supabase Integration
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_KEY=your-anon-key

# Optional: API Configuration
HERMES_PORT=8080
HERMES_HOST=0.0.0.0

Press Ctrl+X, then Y, then Enter to save.

4.4 Start Hermes

hermes agent start

Or run directly:

python -m hermes.agent --config .env
✅ Success! Hermes should now be running on http://localhost:8080
Phase 5

Test Your Setup

5.1 Test Ollama API

curl http://localhost:11434/api/generate -d '{
  "model": "qwen2.5:0.5b",
  "prompt": "Say hello in one word"
}'

5.2 Test Hermes Endpoint

curl http://localhost:8080/api/chat -H "Content-Type: application/json" -d '{
  "message": "What is the capital of France?",
  "model": "qwen2.5:1.5b"
}'

5.3 Check System Status

ollama list
ps aux | grep -E "ollama|hermes"
free -m  # Check available RAM
Phase 6

Optional: Enable XMR Mining

6.1 Install XMRig

pkg install xmrig
xmrig --version

6.2 Configure Mining Pool

cd ~
curl -o config.json -L https://raw.githubusercontent.com/xmrtdao/mmlauncher/main/configs/pool-config.json
nano config.json

Update your wallet address in the config file.

6.3 Start Mining

xmrig -c config.json
💡 Pro Tip: Run mining in a separate Termux session to keep Hermes agents running uninterrupted.

🔧 Troubleshooting

Common Issues

  • Ollama won't start: Check if port 11434 is in use: netstat -tlnp | grep 11434
  • Out of memory: Use smaller models (qwen2.5:0.5b or llama3.2:1b)
  • Pip install fails: Run pkg install rust then retry
  • Hermes 404 errors: Ensure Ollama is running: ollama serve &
  • Permission denied: Run termux-setup-storage and grant permissions

📋 Quick Reference

Essential Commands

# Activate Hermes environment
cd ~/hermes-env && source venv/bin/activate

# Start Ollama
ollama serve &

# Start Hermes
hermes agent start

# Check running processes
ps aux | grep -E "ollama|hermes"

# View Ollama models
ollama list

# Stop all services
pkill -f ollama
pkill -f hermes

Useful Shortcuts

  • Ctrl+C — Stop current process
  • Ctrl+Z — Background process
  • bg — Resume backgrounded process
  • fg — Bring background process to foreground
  • exit — Close Termux session
